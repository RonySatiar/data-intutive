A tool for developing modular data workflows

Viash is a tool for developing modular data workflows, enabling reproducible and reusable computational research with less IT overhead. It empowers scientists to create, reuse, and adapt standardized pipeline building blocks and integrate them into their data pipelines. This cutting-edge approach that provides long-term advantages to your biotech research.

Separating IT operations from research

Separating IT operations from research allows scientists to concentrate only on their studies without dealing with technical challenges such as reproducibility, scalability, and API management. Moreover, by distinguishing research tool development from pipeline development, we promote better tool reuse, continuous improvement, and more effective collaboration among researchers.

Viash ensures your projects remain vendor-independent(1)

We help researchers focus on core scientific inquiries by untangling the intricacies of IT operations. Viash ensures your projects remain vendor-independent(1) , facilitating collaboration and adaptability across various systems and technologies.

Viash integrates effortlessly with Nextflow(1) (2)

As a trusted, industry-ready, and open-source solution, Viash is used by leading biotech entities worldwide.

NoPipeline in Biotech R&D

In the long term, it’s not only the pipeline.

Tactical

  • Decoupling IT operations from research knowledge enables researchers to focus solely on scientific inquiries, free from complexities like reproducibility, scalability, and API management.
  • Viash simplifies the creation of Nextflow data pipelines by turning scripts into easily-integrated and individually executable modular building blocks.
  • Decoupling individual research tool development from pipeline development enhances reuse, improvement, and fosters efficient collaboration among researchers.
  • Viash’s modules facilitate flexible data pipeline design and development, allowing for easy adaptation to changing needs and enabling experimentation in a dynamic R&D environment.

Strategical

  • Modularity provides the ability to implement a long-term data workflow strategy for effectively managing and reusing knowledge.
  • A modular approach to data workflows leads to improved efficiency in the use of research staff and IT support, resulting in a higher-quality end product.
  • The use of code generation leads to standardized and validated high-quality software code and Nextflow modules to ensure the reliability and accuracy of biotech processes.
  • Using Viash prevents vendor lock-in(1):, ensuring long-term adaptability and sustainability for your R&D knowledge and organization.

Operational

  • Viash supports the combination of multiple scripting languages (Bash, Python, R, Scala, JS, C#) in a pipeline.
  • Viash generates standalone, Docker, and Nextﬂow executables to be run independently of the script’s language, using metadata as a basis.
  • Viash includes built-in unit testing capabilities for debugging and testing of individual components in a pipeline.
  • Viash executables include extra support for developers through command-line arguments, type checking, and validation of required parameters and input ﬁles.
